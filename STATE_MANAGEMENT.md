# State Management

The app uses Redux Toolkit for state and redux-saga for side effects. The production store is defined in `src/app/store/store.ts`.

## Store Setup

```ts
export const rootReducer = combineReducers({
  catalog: catalogReducer,
  cart: cartReducer,
  checkout: checkoutReducer
});
```

`RootState` is inferred from `rootReducer`. Components must use:

- `useAppDispatch`
- `useAppSelector`

Thunk is disabled:

```ts
getDefaultMiddleware({ thunk: false }).concat(sagaMiddleware);
```

Serializable checks remain enabled. Do not disable them globally.

## Global State Shape

### `catalog: CatalogState`

```ts
{
  ids: ProductId[];
  entities: Record<ProductId, Product>;
  categories: Category[];
  page: number;
  limit: number;
  hasMore: boolean;
  total: number;
  query: string;
  debouncedQuery: string;
  selectedCategoryId: CategoryId | null;
  sort: ProductSort;
  status: AsyncStatus;
  error: string | null;
  activeRequestSignature: CatalogRequestSignature | null;
  productDetailsStatusById: Partial<Record<ProductId, AsyncStatus>>;
  productDetailsErrorById: Partial<Record<ProductId, string | null>>;
}
```

`AsyncStatus` is:

```ts
'idle' | 'loading' | 'refreshing' | 'loadingMore' | 'success' | 'error';
```

Catalog list state and product detail lifecycle are intentionally separate. A failed detail lookup must not set catalog list `status` to `error`.

### `cart: CartState`

```ts
{
  itemsByProductId: Record<ProductId, CartItem>;
  hydrationStatus: 'idle' | 'loading' | 'success' | 'error';
  hydrationError: string | null;
}
```

Persistable cart state is:

```ts
Pick<CartState, 'itemsByProductId'>;
```

Only `itemsByProductId` is saved to AsyncStorage. Hydration status/error fields are runtime-only.

### `checkout: CheckoutState`

```ts
{
  status: 'idle' | 'loading' | 'success' | 'error';
  latestOrderId: OrderId | null;
  error: string | null;
}
```

Checkout state is reset from success/error screens before returning to catalog or checkout.

## Root Saga

`src/app/store/rootSaga.ts` forks the feature sagas:

```ts
yield all([fork(catalogSaga), fork(cartSaga), fork(checkoutSaga)]);
```

Keep this `fork` pattern so watcher tasks are isolated.

## Catalog Workflow

### Initial Load

`useCatalog` dispatches:

```ts
catalogActions.catalogRequested('initial');
```

`catalogRequested` sets:

- `status: 'loading'`
- `activeRequestSignature` based on `{ page, limit, query, categoryId, sort }`

`catalogSaga` selects the current request, calls `ProductRepository.listProducts`, and dispatches `catalogPageLoaded`.

### Search Debouncing

Search input dispatches:

```ts
searchQueryChanged(query);
```

`catalogSaga` debounces this action for `350ms`:

```ts
yield debounce(350, catalogActions.searchQueryChanged.type, commitDebouncedSearch);
```

After debounce:

1. `debouncedSearchCommitted(query)` stores the committed query and resets page/hasMore.
2. `catalogRequested('initial')` starts a fresh list request.

### Category And Sort Changes

`categoryChanged` and `sortChanged` update filter state and reset pagination. The saga listens with `takeLatest` and dispatches `catalogRequested('initial')`.

### Infinite Scroll

`ProductGrid` calls `onLoadMore` from `FlashList.onEndReached`.

`useCatalog.loadMore` guards against duplicate loads:

- no request if `hasMore` is false
- no request while status is `loading`, `refreshing`, or `loadingMore`

When valid, it dispatches:

```ts
catalogRequested('loadMore');
```

The saga requests `page + 1`. The reducer appends products for `loadMore` and replaces products for `initial`/`refresh`.

### Stale Response Guard

Every list request gets a `CatalogRequestSignature` generated by `createCatalogRequestSignature`.

`catalogPageLoaded` and `catalogFailed` are ignored unless their `requestSignature` matches `state.activeRequestSignature`. This protects search/filter/pagination from stale or out-of-order responses.

### Product Details

`useProductDetails(productId)` dispatches:

```ts
productDetailsRequested(productId);
```

If the product already exists in `catalog.entities`, the reducer marks detail status as `success`.

If not, `catalogSaga` calls `ProductRepository.getProductById(productId)`:

- product found: `productDetailsLoaded(product)`
- null result: `productDetailsFailed({ productId, error: 'Product not found.' })`
- thrown error: `productDetailsFailed({ productId, error })`

The detail screen shows loading for `idle/loading`, an error/not-found state for missing products, and never relies on catalog list status for detail failures.

## Cart Workflow

### Hydration

`store.ts` dispatches `cartHydrationRequested()` immediately after starting `rootSaga`.

`cartSaga` calls `CartPersistenceRepository.loadCart`.

Success:

```ts
cartHydrated(cart);
```

Failure:

```ts
cartHydrationFailed(message);
```

Invalid persisted shapes are sanitized by `CartPersistenceRepository.parseCartState`.

### Add To Cart

`ProductDetailsScreen` dispatches:

```ts
addToCart({ product, quantity, addedAt: CartRepository.now() });
```

`addToCart` intentionally receives the full `Product` because it clamps against `product.stock.available` at add time with `clampCartQuantity`.

### Increment And Decrement

Increment/decrement commands are ID-based:

```ts
incrementCartItem({ productId });
decrementCartItem({ productId });
```

`incrementCartItem` is a command action. The reducer does not mutate state directly. `cartSaga` selects the cart item and current catalog product, then dispatches:

```ts
cartItemIncremented({ productId, maxQuantity: product.stock.available });
```

The reducer clamps to `maxQuantity`.

`decrementCartItem` mutates directly and removes the item when quantity reaches zero.

### Persistence

Cart persistence is debounced for `250ms` after:

- `addToCart`
- `cartItemIncremented`
- `decrementCartItem`
- `removeCartItem`
- `cartCleared`

The saga selects `selectPersistableCartState`, so only `itemsByProductId` is passed to `CartPersistenceRepository.saveCart`.

Save failures are caught and logged. They must not kill the watcher.

## Checkout Workflow

`CheckoutScreen` calls `placeOrderRequested`.

`checkoutSaga` selects:

- `selectCartItems`
- `selectCartPricing`

If the cart is empty, it dispatches:

```ts
checkoutFailed('Add at least one item before checkout.');
```

Otherwise it calls `OrderRepository.placeOrder({ items, pricing })`.

Success:

1. `orderPlaced(order)`
2. `cartCleared()`
3. `CheckoutScreen` navigates to `OrderSuccess`

Failure:

1. `checkoutFailed(message)`
2. `CheckoutScreen` navigates to `OrderError`
